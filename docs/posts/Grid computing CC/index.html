Introduction

Grid computing is the collection of computer resources from multiple locations to reach a common goal. The grid can be thought of as a distributed system with non-interactive workloads that involve a large number of files.

Grid computing is distinguished from conventional high-performance computing systems such as cluster computing in that grid computers have each node set to perform a different task/application. Grid computers also tend to be more heterogeneous and geographically dispersed (thus not physically coupled) than cluster computers.

Although a single grid can be dedicated to a particular application, commonly a grid is used for a variety of purposes. Grids are often constructed with general-purpose grid middleware software libraries. Grid sizes can be quite large.

Features of Grid Computing

They enable the ability of virtualization of distributed computing and data resources such as processing, network bandwidth and storage capacity to create a single system image, granting users and applications seamless access to vast IT capabilities.

Both the distributed and the grid computing looks similar at the first glance, but there are many differences between them. The main difference between distributed and grid computing is that the distributed computing helps to share the workload between several autonomous computers to accomplish a task while the grid computing connects several distributed computing systems to coordinate to solve a single problem.

Grid Computing Architecture

They provide dependable, consistent, pervasive, and inexpensive access to high-end computational capabilities.

They are composed of a collection of distributed, possibly heterogeneous resources, which are coordinated to present users with a single, integrated, and unified resource.

Users (or the client) submit jobs to the system, which are scheduledb byt the system. The user will not know where the job is executed. To get the result the user will hook into the grid and get the result.

The Grid links together computing recsources such as PCs, workstations, servers, storage systems, instruments and sensors into a unified resource. The system then executes the jobs on the resources and returns the results to the users.

Advantages of grid computing

  • Effitient use of resources.
  • Can share more information and move huge amounts of data easily
  • Can run the application in a dynamic virtualized environment and can be accessed by everyone who is having the authorization
  • Interaction with the underneath layers are transparent and seamless to the user
    • Here underneath layers means the hardware and software layers

What is the need for grid computing

  • Computer simulations & modelling are more cost effective than experimental methods
  • Engineering problems are becoming more complex and users need more accurate and precise solution to their problems in shortest possible time which make grid computing a charm where distributed computing fails
  • Data vizualization and exploiting under utilized resources will be possible with grid computing
